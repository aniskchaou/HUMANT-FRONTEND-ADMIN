Documentation: Installation Guide

Installation is divided into two tracks: Local Installation and Remote Installation.

Back to Documentation
Track A: Local Installation

Use this for development, testing, and first-time setup on your own machine.

Local
Step 1: Install Prerequisites
Required Software
  • Node.js 14.x or 16.x
  • npm
  • Java 11 JDK
  • PostgreSQL 12+
Recommended Tools
  • Git
  • pgAdmin or another PostgreSQL client
  • Terminal access with admin permissions
Step 2: Local Installation
  1. Clone project and move into root folder.
  2. Run npm install.
  3. Create PostgreSQL database named humant.
  4. Update datasource values in backend/src/main/resources/application.properties.
  5. Start backend from backend folder.
  6. Start frontend on port 4201.
  7. Open app and verify login plus module navigation.
Reference Commands
Project Setup
git clone <repository-url>
cd humant
npm install
Database
CREATE DATABASE humant;
Backend (Windows)
Push-Location .\backend
.\mvnw.cmd spring-boot:run
Backend (Linux/macOS)
cd backend
./mvnw spring-boot:run
Frontend
npm run serve -- --port 4201

Verify endpoints: frontend at http://localhost:4201 and backend at http://localhost:8080.

Track B: Remote Installation (Server/Cloud)

Use one of the following options to deploy HUMANT remotely.

Step 3: Choose Deployment Target
AWS Deployment
  1. Provision RDS PostgreSQL instance.
  2. Deploy backend to EC2, Elastic Beanstalk, or ECS.
  3. Deploy frontend to S3 + CloudFront.
  4. Use ALB or Nginx for API routing and TLS.
  5. Store secrets in AWS Systems Manager Parameter Store.
  6. Enable CloudWatch alarms and backup policies.
Heroku Deployment
  1. Create separate apps for backend and frontend static host.
  2. Add Heroku Postgres addon for database.
  3. Set backend env vars for DB URL/user/password.
  4. Deploy backend JAR via GitHub integration or Heroku CLI.
  5. Publish Angular build to static host and point API base URL.
  6. Validate login flow and role-based route access.
VPS Deployment
  1. Install Java 11, PostgreSQL, Nginx, and Node.js.
  2. Build frontend and backend artifacts.
  3. Run backend as systemd service.
  4. Serve frontend with Nginx and proxy API to backend.
  5. Configure firewall, TLS certificates, and backups.
  6. Run smoke tests after each deployment.
Recommended: keep backend private to internal traffic and expose only reverse-proxy endpoints publicly.
Complete VPS Deployment Steps (Ubuntu)
  1. Install packages: Nginx, Java 11, PostgreSQL, certbot.
  2. Create PostgreSQL database and dedicated DB user.
  3. Build project artifacts (frontend + backend JAR).
  4. Upload backend JAR to /opt/humant/backend/app.jar.
  5. Upload frontend build to /var/www/humant.
  6. Create and enable humant-backend.service.
  7. Configure Nginx for Angular SPA and API proxy.
  8. Enable TLS and firewall rules, then run smoke tests.
Key Commands
sudo apt update
sudo apt install -y nginx postgresql postgresql-contrib openjdk-11-jre-headless certbot python3-certbot-nginx
sudo systemctl enable humant-backend
sudo systemctl restart humant-backend
sudo nginx -t ; sudo systemctl reload nginx
sudo certbot --nginx -d your-domain.example.com
Complete AWS Deployment Steps
  1. Create RDS PostgreSQL instance and secure access from app layer only.
  2. Deploy backend artifact to EC2, ECS, or Elastic Beanstalk.
  3. Set datasource environment variables with RDS endpoint.
  4. Build frontend and upload static files to S3.
  5. Put CloudFront in front of S3 and configure SPA fallback.
  6. Configure API routing via ALB or reverse proxy.
  7. Enable ACM certificates, CloudWatch logs, alarms, and backups.
Key Commands
npm ci
npm run build
cd backend ; ./mvnw clean package
aws s3 sync dist/ s3://your-frontend-bucket --delete
Complete Heroku Deployment Steps
  1. Create Heroku app for backend and add Heroku Postgres.
  2. Set Java buildpack and backend datasource config vars.
  3. Deploy backend and watch logs for startup success.
  4. Deploy frontend to static hosting and point API URL to Heroku backend.
  5. Validate login, data read/write, and role-based route access.
Key Commands
heroku login
heroku create humant-backend
heroku addons:create heroku-postgresql:essential-0 -a humant-backend
heroku config:set SPRING_DATASOURCE_URL=<jdbc-url> -a humant-backend
heroku config:set SPRING_DATASOURCE_USERNAME=<db-user> -a humant-backend
heroku config:set SPRING_DATASOURCE_PASSWORD=<db-password> -a humant-backend
git push heroku main
Step 4: User Panel Details

User Panel behavior is role-based. Each logged-in user sees only modules allowed by access policy and assigned authority.

Panel Sections
  • Executive Dashboard
  • People and Structure modules
  • Talent and Growth modules
  • Time and Compensation modules
  • Risk, Exit, Settings, and Documentation
Role Visibility (Typical)
  • Admin: full workspace visibility
  • HR: people, talent, and compliance modules
  • Payroll/Finance: salary, payslip, loan, expenses, advances
  • Manager: team operations and review-related modules
  • Employee: self-service modules (profile, leave, payslip, expense)
Step 5: Go-Live Checklist
Security
  • Strong DB and service credentials configured
  • TLS certificates installed and auto-renewal validated
  • Least-privilege access to deployment hosts
  • Sensitive modules restricted by policy and role
Operations
  • Health checks and alerting enabled
  • Database backup and restore test completed
  • Rollback procedure documented and validated
  • User login and role-routing smoke test completed
Troubleshooting
  • DB connection failed: verify PostgreSQL service, credentials, and database name.
  • Frontend not loading: re-run npm install and use Node.js 14/16.
  • Port conflict: change frontend port or stop process on 4201/8080.
  • Access denied after login: verify role assignment and access policy rules.
